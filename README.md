项目：Boss直聘职位爬虫
简介：可以爬取特定城市特定搜索词的boss直聘岗位分析，方便对岗位就业条件与薪资待遇水平等分析，暂时只通过词云进行了技能关键词分析，有想法的朋友可以自己拓展分析数据。
      代码通过DP浏览器获取cookies并通过定时滑动点击等操作更新cookies（因为一个cookies仅支持进行4-5次详情获取），列表爬取和详情爬取则是通过boss直聘的api来获取。


功能：支持关键词搜索，支持特定城市搜索（默认为杭州）
      DrissionPage浏览器模拟浏览器请求有效获取cookies并防止被封
      修改浏览器指纹使得DrissionPage不被识别出（主要修改了浏览器返回内容，DP默认浏览器返回内存为空会被识别出爬虫）
      数据可导出为CSV并用清洗代码清洗
      设置心跳系统定时滑动点击刷新cookies
      附带词云构建代码，可以简单分析岗位需求。（部分停用词为哈工大中文停用词表，部分为作者自己手工输入）


使用方法：下载除requirement.txt的所有文件并放在同一文件夹下即可使用默认配置
          requirement.txt中是作者的调用库的版本，如果能够直接使用可以不管


修改配置方法：第一步：查看spider代码，其中初始的URL为boss直聘网站拼接了城市id和搜索词，是用于获取cookies的搜索页，其中城市id可以自己去boss直聘网站获取。
              第二步：修改完初始URL链接和关键词后修改get_cookie_headers()、api_job_list()两个函数中的current_headers和data_str配置，主要修改链接中的城市id，关键词在代码开头的config中统一修改。




作者说：因为技术力有限而且cookies过期很快所以选择了使用DP获取cookies并定时刷新的策略来避开逆向部分，除开逆向外boss直聘也算是我遇到第一个反爬很严格的网站了，
第一次遇见能够识别无头浏览器的网站，最开始给我造成了不少麻烦也让我了解到了一些浏览器指纹的知识；禁止devtools\f12不让看html文件也让我大开眼界（解决方法是使用
dp保存html后禁止html的js跳转功能），不过使用reqable后发现有接口后就直接用接口获取数据了，解析json数据比解析html方便很多。
